# Steve Rude's Opinions

- Prefer Go(lang) as programming language.
- Prefer Svelte 5 for frontend.
- Prefer Mittsu (mittsu.dev) for UI.
- Prefer simple over complex solutions.
- Write a minimal amount of tests always, but don't create vanity tests.
- Use A3T technology where possible.
- Always research and plan first.
- Validate all work.
- Be complete, don't create stub functions and fake implementations.
